## ✨ این ابزار چیست؟

**Proxy Pro Advanced** یک ابزار حرفه‌ای خط فرمان است که لینک‌های خام پروکسی را به یک لیست **تمیز، تست‌شده، مرتب‌شده و آماده‌ی استفاده** تبدیل می‌کند.
کافی است لینک، فایل یا هر متنی به آن بدهید — خودش جمع‌آوری می‌کند، تکراری‌ها را حذف می‌کند، تست TCP می‌گیرد، پروتکل را تشخیص می‌دهد، تأخیر را اندازه می‌گیرد، ناشناسی و کشور را پیدا می‌کند و همه چیز را در پوشه‌های مرتب و شماره‌گذاری‌شده ذخیره می‌کند.

> روی ویندوز، مک و لینوکس اجرا می‌شود. بدون اکانت. بدون کلود. ۱۰۰٪ لوکال.

## 🚀 امکانات ویژه

| قابلیت | توضیح |
|---|---|
| 🧠 **جمع‌آوری هوشمند** | از URL، فایل، پوشه یا حتی کپی‌پیست مستقیم (`ip:port` یا `ip:port:user:pass`) |
| 🔒 **منابع خصوصی (Private)** | منابعی که یوزر و پسورد اختصاصی دارند — کاملاً از منابع عمومی جدا نگه داشته می‌شوند |
| ⚡ **تست دومرحله‌ای** | مرحله ۱: تست دسترسی TCP → مرحله ۲: تست کامل پروتکل (http/https/socks4/socks5) |
| 🕵️ **تشخیص ناشناسی** | جدا کردن Transparent / Anonymous / Elite |
| 🌍 **موقعیت جغرافیایی** | کشور و ISP برای هر پروکسی |
| 🔑 **پروکسی‌های Auth** | یوزر و پسورد کامل داخل `alive_with_auth.txt` ذخیره می‌شود |
| 💾 **تاریخچه‌ی شماره‌گذاری‌شده** | هر بار Collect یا Test پوشه‌ی جدید می‌سازد — هیچ فایلی بازنویسی نمی‌شود |
| ▶️ **ادامه‌ی خودکار (Resume)** | با `Q` یا `Ctrl+C` متوقف کنید، دفعه‌ی بعد از همانجا ادامه می‌دهد |
| 🎨 **رابط مدرن** | پنل‌های رنگی، پیشرفت زنده، نوار وضعیت بنفش/فیروزه‌ای |

## 📦 نصب

### ۱️⃣ نصب Node.js (یک‌بار)
از [nodejs.org](https://nodejs.org) نسخه‌ی **۱۸ به بالا** را دانلود و با تنظیمات پیش‌فرض نصب کنید.

### ۲️⃣ دانلود ابزار
فایل ZIP را از بخش [Releases](../../releases) بگیرید و در هر جا اکسترکت کنید.

### ۳️⃣ نصب پیش‌نیازها
**ویندوز:** روی `INSTALL_FIRST.bat` دابل‌کلیک کنید.
**مک / لینوکس:**
```bash
npm install
```

### ۴️⃣ اجرا
**ویندوز:** روی `RUN.bat` دابل‌کلیک کنید.
**مک / لینوکس:**
```bash
node proxy-pro.js
```

## 🎛️ منوی اصلی

بالای صفحه نوار وضعیت زنده را می‌بینید:

```
Mode : balanced   Two-Phase: on   Conc: 150   Anon: on   Geo: on
```

| کلید | گزینه | کاربرد |
|:---:|---|---|
| `1` | **Add sources** | افزودن منابع Public / Private / Smart Paste |
| `2` | **Manage sources** | مشاهده و حذف منابع ذخیره‌شده |
| `3` | **Collect proxies** | جمع‌آوری از منابع (زیرمنو در ادامه) |
| `4` | **Test proxies** | تست آخرین لیست جمع‌آوری‌شده (اول مود می‌پرسد) |
| `5` | **Settings / Modes** | تغییر مود تست و تنظیمات دستی |
| `6` | **Open output folder** | باز کردن پوشه‌ی خروجی (alive / best / csv / گزارش HTML) |
| `H` | **Help** | راهنمای کامل داخل برنامه |
| `0` | **Exit** | خروج تمیز |

## ➕ افزودن منابع (منوی ۱)

| کلید | نوع | توضیح |
|:---:|---|---|
| `1` | **Public** | URL یا فایل بدون یوزر/پسورد |
| `2` | **Private** | URL/فایل + یوزر و پسورد اختصاصی همان منبع |
| `3` | **Smart Paste** | هر چیزی را پیست کنید — URL، فایل، `ip:port`، `ip:port:user:pass` — خودش دسته‌بندی می‌کند |
| `0` | برگشت | |

می‌توانید چند فایل را با هم Drag & Drop کنید یا یک پوشه کامل بیندازید (همه‌ی `.txt` / `.list` / `.csv` داخلش خوانده می‌شوند).

## 📥 جمع‌آوری پروکسی (منوی ۳)

| کلید | منبع | توضیح |
|:---:|---|---|
| `1` | **Public sources** | فقط URL/فایل‌هایی که در بخش Public اضافه کرده‌اید |
| `2` | **Private sources** | فقط منابعی که یوزر:پس دارند (کاملاً جدا) |
| `3` | **Smart Paste** | فقط پروکسی‌هایی که با Smart Paste اضافه کرده‌اید |
| `4` | **Auto (built-in)** | استفاده از لیست پیش‌فرض داخلی — چیزی در منابع شما ذخیره نمی‌شود |
| `5` | **Both (merged)** | Public + Smart Paste و سپس Auto. **منابع Private عمداً وارد نمی‌شوند** — برای آن‌ها گزینه‌ی ۲ را بزنید. |
| `0` | برگشت | |

---

## 🧪 تست پروکسی‌ها (منوی ۴)

قبل از هر تست از شما پرسیده می‌شود:

| کلید | انتخاب |
|:---:|---|
| `1` | استفاده از **مود فعلی** تنظیمات |
| `2` | **انتخاب خودکار** بر اساس حجم لیست |
| `3` | **انتخاب دستی** از ۸ مود زیر |
| `0` | انصراف |

### مودهای تست

| مود | مناسب برای | Conc | ویژگی |
|---|---|:---:|---|
| **ultra** | لیست‌های ۱۰۰k+ | 400 | بیشترین سرعت، بدون anon/geo |
| **fast** | لیست‌های بزرگ | 250 | ۱ endpoint + geo |
| **balanced** ⭐ | پیش‌فرض | 150 | ۲ endpoint + anon + geo |
| **accurate** | کیفیت بالا | 80 | ۳ endpoint، double-check |
| **deep** | کمترین false-dead | 50 | ۸ endpoint، تأیید مجدد |
| **strict** | فقط بهترین‌ها | 60 | آستانه‌ی خیلی سخت‌گیرانه |
| **lowpc** | سیستم/نت ضعیف | 30 | سبک |
| **sample** | تست سریع | 120 | فقط ۱۰۰۰ تای اول |

هر وقت خواستید با `Q` یا `Ctrl+C` متوقف کنید — نتایج جزئی ذخیره می‌شوند و دفعه‌ی بعد قابل ادامه هستند.

## 📁 ساختار خروجی (هیچ فایلی بازنویسی نمی‌شود)

```
output/
├── All_proxy/
│   └── All_proxy_N/
│       ├── all_proxies.txt        ← همه‌ی پروکسی‌های جمع‌آوری‌شده
│       ├── ip_port_only.txt       ← فقط ip:port
│       ├── http.txt / https.txt / socks4.txt / socks5.txt / unknown.txt
│       └── collect_summary.txt
└── Result_test/
    └── results_N/
        ├── alive.txt              ← پروکسی‌های سالم بدون یوزر:پس (بهترین → ضعیف‌ترین)
        ├── alive_with_auth.txt    ← پروکسی‌های سالم همراه با یوزر:پس کامل
        ├── best.txt / elite.txt   ← فقط بهترین‌ها
        ├── dead.txt
        ├── results.csv / summary.txt / tested_results.json / report.html
        ├── by_type/               ← http / https / socks4 / socks5
        └── by_country/            ← US.txt / DE.txt / IR.txt ...
```

> 💡 در `alive_with_auth.txt` یوزر و پسورد به‌صورت **کامل و بدون ستاره** نوشته می‌شود — فایل روی سیستم خودتان است.

## 🩺 رفع اشکال

<details>
<summary><b>وقتی عدد می‌زنم منو هیچ واکنشی نشان نمی‌دهد</b></summary>

در **v1.0.7** برطرف شد — مشکل focus ورودی در بعضی ترمینال‌های ویندوز رفع شده است. اگر باز هم دیدید، از `cmd.exe` یا Windows Terminal اجرا کنید و بعد از عدد Enter بزنید.
</details>

<details>
<summary><b>Auto-collect (گزینه ۴) پروکسی‌های Private را هم می‌آورد</b></summary>

نه — از نسخه‌ی **v1.0.5** به بعد گزینه‌های `4` و `5` **هرگز** به منابع Private دست نمی‌زنند. برای اجرای Private از گزینه‌ی `2` استفاده کنید.
</details>

<details>
<summary><b>همه‌ی پروکسی‌ها Dead نشان داده می‌شوند</b></summary>

مود را عوض کنید (`deep` یا `accurate`)، اینترنت/VPN را چک کنید، یا از Settings → Test URLs آدرس‌های تست را عوض کنید.
</details>
